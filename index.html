// the whole idea

One model has one opinion.
Two models have an argument.

A single agent commits to its first idea and runs. Tales makes two strong models plan and execute in parallel, then cross-check each other before anything touches your code. You get the disagreement before the diff, not after.

drafter

One model proposes the approach and writes the plan.

⇄
critic

The other pokes holes, asks the questions, names the trade-offs.

→
you

You read the debate and pick the executor. Hard gate — nothing auto-runs.

Not a hunch — the field keeps re-discovering it. OpenRouter ships model fusion (blend several models for a stronger answer); AWS Kiro splits spec-then-build across agents. Tales does it where you already work — your terminal — and keeps a human on the trigger.

// not just two — and not just the expensive ones

Mix the room.
Tier the cost.

Tales isn't locked to two models or one vendor. Bring any AI coding CLI you trust — connect two, connect three, mix vendors. Each tool plugs in behind a small adapter, so the room is whatever you make it.

And the room doesn't have to be uniform. Run your smart, expensive models on the part that actually matters — the plan and the argument — then hand the agreed plan to a cheaper, faster model to implement. You pay top dollar for judgment, not for typing.

bash — tiered run 
# two strong models plan + argue, a cheap/fast one implements
tales run "add OAuth login" \
  --drafter <tool-a> --drafter-model <strong> \
  --critic  <tool-b> --critic-model  <strong> \
  --execute <tool-c> --execute-model <cheap-fast>

Live today in tales run via --execute-model. The core is already model- and tool-agnostic; bigger rooms (N planners), more adapters, and same-tool tiering are what the CLI is growing into next.

// run it

Type tales. That's the whole UI.

No dashboard, no login, no accounts. Connect the CLIs you already have and go.

1
connect

Pick the AI coding tools to put in the room — any CLI you already use. Bring two; bring three.

2
plan default

They discuss in a live chat. Type to interject any time — you're a participant, not a spectator.

3
pick

They recommend an executor. You confirm, override, or reject. This gate cannot be skipped.

4
execute

The chosen tool builds the plan in an isolated git worktree. You get a clean, reviewable diff.

// quickstart

Up in two minutes.

bash 
# build it (needs Rust + your AI coding CLIs on PATH)
git clone https://github.com/cesarfrancots/tales && cd tales
cargo build --release

# open the terminal — connect your tools, then plan
tales

# or go straight in, scriptable:
tales run "add OAuth login" --drafter <tool-a> --critic <tool-b> --execute <tool-a>

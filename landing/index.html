How it works

Four steps, one rule:
you always confirm.

Tales keeps you in the loop at every stage. The AIs do the thinking; you do the deciding.

1
Discuss
Claude Code and Codex read your task and debate the best approach in a shared chat thread — surfacing trade-offs before a single line is written.
2
Recommend
The agents reach consensus and nominate which one should execute, with a rationale. No hidden decisions.
3
You decide
You review the plan and the recommendation. Approve, redirect, or cancel — nothing runs without your explicit go-ahead.
4
Execute
The chosen agent carries out the work in a sandboxed git worktree. The other agent reviews the output. You get a clean diff.

Features

Built for trust,
not speed-running.

Every design decision in Tales is oriented around keeping humans meaningfully in control of agentic AI.

🤝
Multi-agent collaboration
Claude Code and Codex work together as a team — each with different strengths, cross-checking each other's reasoning before making any recommendation.
⚡
Real-time streaming
Watch the discussion unfold live. Streamed token-by-token so you can follow the agents' thinking as it happens, not after the fact.
🔒
Human-in-the-loop gate
A hard confirmation step before any code runs. No opt-out, no auto-approve mode. You remain the final authority on every action.
🌿
Git-worktree isolation
Execution happens in a fresh git worktree — your working directory is never touched speculatively. Inspect the diff before merging anything.
🔌
Pluggable adapters
Swap in different models or AI providers without changing the orchestration logic. The adapter interface is small and well-defined.
📖
Open source, MIT
Fully auditable. No telemetry, no cloud dependency, no black boxes. Read the code, fork it, extend it — it's yours.

Quickstart

Up and running
in two minutes.

Tales runs locally. No accounts — just your own Anthropic and OpenAI API keys.

  • 1

    Clone — grab the repo with git clone.

  • 2

    Build — compile the release binary with cargo build --release.

  • 3

    Set your keys — export ANTHROPIC_API_KEY and OPENAI_API_KEY.

  • 4

    Run — pass your task as an argument and approve the plan when ready.

shell 
# Clone and enter the repo
git clone https://github.com/cesarfrancots/tales
cd tales

# Build the release binary
cargo build --release

# Export your API keys
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...

# Run with your task
./target/release/tales-tui "add OAuth login"

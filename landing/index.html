why two

One model has one opinion.
Two models have an argument.

A single agent commits to its first idea. Tales makes one model draft and another critique before the diff exists, so the assumptions surface while they are still cheap.

Claude CodeDRAFTER

Proposes the concrete path, names files, keeps the plan executable.

CodexCRITIC

Finds edge cases, calls out risk, and asks the question the first pass skipped.

YouGATE

Approve, override, or reject. The executor does not run until you choose.

The point is not autonomy theater. It is a local terminal that puts disagreement in front of you before any tool starts changing files.

tier the cost

Use expensive judgment.
Use cheap tokens for typing.

Put your strongest models on planning and critique, then hand the agreed plan to a cheaper, faster executor when the work is mechanical.

bash - tiered run 
# strong models debate, a fast model implements
tales run "add OAuth login" \
  --drafter claude --drafter-model opus \
  --critic  codex  --critic-model  gpt-5-codex \
  --execute gemini --execute-model gemini-2.5-flash

run it

Type tales.
That is the whole UI.

No dashboard, no account, no hidden daemon. Connect the CLIs you already have and keep the trigger.

01connect

Pick Claude Code, Codex, Open Code, Gemini, GLM, Kimi, or another CLI row.

02preflight

See profile memory, prompt forecast, cache state, and tool recommendations.

03plan

They discuss in a live terminal. You can interject.

04pick

The recommendation appears behind a hard approval gate.

05execute

The chosen tool builds the agreed plan and returns a diff.

quickstart

Up in two minutes.

bash 
# build it
git clone https://github.com/cesarfrancots/tales && cd tales
cargo build --release

# local preflight, memory, and evals
tales doctor --all
tales profile refresh
tales eval compare "fix lint errors"

# open the terminal
tales
